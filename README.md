### Flujo de trabajo GitFlow

GitFlow es una estrategia de ramificaci√≥n en Git dise√±ada para optimizar la colaboraci√≥n en equipos y mantener un control eficiente de versiones. Este enfoque establece ramas espec√≠ficas con roles bien definidos, permitiendo integrar nuevas funcionalidades, gestionar lanzamientos y resolver incidencias de forma ordenada.


### Ramas principales

- **main (principal)**  
  Contiene el c√≥digo estable y listo para producci√≥n, correspondiente a las versiones oficiales de Trustora.  
  Cada release es marcado con **etiquetas sem√°nticas** (ejemplo: `v1.0.0`) para facilitar el rastreo, la documentaci√≥n y la aplicaci√≥n de actualizaciones.

- **develop (rama de desarrollo)**  
  Alberga la versi√≥n m√°s reciente en estado de preproducci√≥n, donde se integran todas las funcionalidades completadas.  
  Sirve como base para pruebas internas y ajustes antes de fusionar los cambios a la rama principal.



### Ramas de soporte

- **feature/* (ramas de caracter√≠sticas)**  
  Cada nueva funcionalidad se desarrolla en una rama independiente (ejemplo: `feature/login`) derivada de `develop`.  
  Permiten trabajo paralelo sin afectar el c√≥digo principal hasta su revisi√≥n y aprobaci√≥n.  
  Una vez completada, se fusiona nuevamente con `develop`.

- **release/* (ramas de lanzamiento)**  
  Se crean para preparar versiones espec√≠ficas del sistema.  
  Facilitan pruebas finales, correcci√≥n de errores menores y ajustes previos antes de desplegar en `main`.

- **hotfix/* (ramas de emergencia)**  
  Atienden errores cr√≠ticos detectados en producci√≥n.  
  Estas ramas derivan directamente de `main` y, una vez resuelto el problema, se fusionan tanto en `main` como en `develop`, asegurando coherencia entre ambas ramas.



### Beneficios del modelo aplicado

- Control claro del ciclo de vida de cada funcionalidad.  
- Integraci√≥n continua sin comprometer la estabilidad del producto.  
- Documentaci√≥n clara mediante etiquetas sem√°nticas.

- Escalabilidad del equipo con trabajo paralelo organizado.
